# 로컬 통합 개발 환경(DB + 백엔드 + 프론트엔드) 구축 결과

본 문서는 Docker 기반 MariaDB와 백엔드(Spring), 프론트엔드(React) 서버를 동시에 구동하여 연동 테스트를 완료한 결과 보고서입니다.

## 🛠️ 작업 내용 및 결과

### 1. MariaDB 컨테이너 세팅 및 우회
- 기존 로컬 환경에 포트 `3306`을 점유 중인 프로세스(네이티브 MariaDB/MySQL 데몬)가 발견되어 포트 충돌이 발생했습니다.
- 백엔드 설정(`application.yaml`)의 포트를 `3307`로 변경하고, Docker 컨테이너 구동 시 호스트 포트를 `3307:3306`으로 매핑하여 포트 충돌을 깔끔하게 우회했습니다.
- **결과**: `mariadb-stockgame` 컨테이너 백그라운드 구동 성공 ✅

### 2. Spring Boot 백엔드 구동 (`spring-boot:run`)
- 애플리케이션이 정상적으로 시작되면서, 빈 MariaDB 공간에 DDL(`update`) 옵션에 따라 테이블들이 성공적으로 자동 생성되었습니다.
- `DataInitializer`가 동작하여 주식 기초 데이터와 초기 사용자 정보(더미 데이터)를 DB에 완벽히 삽입(`INSERT`)했습니다.
- **결과**: 8882 포트에서 REST API 및 STOMP(웹소켓) 서버 대기 중 ✅

### 3. React 프론트엔드 의존성 설치 및 구동 (`npm run dev`)
- `package.json` 기반으로 필요한 Node 패키지들을 `npm install` 로 내려받아 설치를 마쳤습니다.
- Vite 개발 서버를 구동했으며 `localhost:5173` 포트에서 정상적으로 서빙을 시작했습니다.
- 앞서 작성한 `vite.config.js` 프록시 서버 설정을 통해, React 페이지 접속 시 `/api/*` 요청이 로컬 8882번(Spring Boot) 포트로 성공적으로 포워딩되는 상태가 되었습니다.
- **결과**: 프론트엔드 - 백엔드 - DB가 하나의 파이프라인으로 연결 성공 ✅

## 🚀 향후 작업
- 이제 브라우저를 켜고 [http://localhost:5173](http://localhost:5173) 로 접속하면 개발된 React SPA 화면을 직접 조작하고, 데이터베이스에 쌓이는 내역을 실시간으로 관찰할 수 있습니다.
- 이 완벽한 샌드박스(개발 환경) 위에서 프론트엔드의 잔여 컴포넌트 마이그레이션 및 캔들스틱 차트 연동을 쾌적하게 진행할 수 있습니다.
